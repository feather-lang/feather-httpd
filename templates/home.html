Welcome to feather-httpd

You're looking at a live HTTP server controlled entirely by Feather scripts. Every route, response, and template render you see is defined in a single .tcl file.

What is this?

This demo shows how to embed Feather in a Go application. The HTTP server is written in Go, but all routing logic and request handling is scripted in Feather—a Tcl-like language designed for embedding.

Why is this cool?

  • Runtime control — modify server behavior without restarting
  • Live introspection — inspect running state like active SSE connections
  • Simple DSL — configure routes and handlers in a readable script
  • Performance where it matters — Go handles the heavy lifting
  • User-scriptable — Feather provides a thin control layer anyone can customize

Try the REPL

Open the web REPL and modify this running server in real-time. Paste this to redefine the About page:

route GET /about {
    template respond about title "About - Modified!"
}

Then visit /about to see your change. You can define new routes, modify existing ones, create templates, and change server behavior—all without restarting.

Try These Endpoints

Visit /chat to see a real-time chat room powered by Server-Sent Events. Or try these from your terminal:

# Dynamic path parameters
curl http://localhost:8080/hello/YourName

# JSON API response
curl http://localhost:8080/api/users/42

# Echo request body
curl -X POST -d '{"message":"hello"}' http://localhost:8080/api/echo

# Query parameters
curl "http://localhost:8080/search?q=foo&limit=5"
